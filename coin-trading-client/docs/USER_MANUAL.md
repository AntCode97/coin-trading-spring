## 앱 개요

Bithumb 암호화폐 거래소에서 자동/수동 트레이딩을 수행하는 데스크톱 앱이다. 세 가지 모드를 지원한다:

| 모드 | 설명 |
|------|------|
| **수동 트레이딩** | 사용자가 직접 마켓을 선택하고 주문을 실행 |
| **오토파일럿** | 3개 엔진(SCALP/SWING/POSITION)이 자동으로 기회 탐지 및 매매 |
| **AI Day Trader** | LLM이 실시간으로 시장을 분석하고 독립적으로 단타 매매 |

앱은 원격 API 서버(Spring Boot)와 통신하며, 차트 데이터/주문/포지션 관리를 서버에 위임한다.

## 메인 워크스페이스

앱 실행 시 나타나는 기본 화면이다. 한 화면에 모든 핵심 정보가 표시되며 스크롤 없이 사용할 수 있다.

**레이아웃 구성:**

```
+--------------------------------------------------+
| 커맨드바 (KPI, 긴급 중지, 메뉴)                    |
+--------+----------------------+------------------+
| 마켓   |      차트 영역        |   액션 콘솔       |
| 사이드  |                      |   (컨트롤/리스크/  |
| 바     |                      |    모니터/고급)    |
|        |                      |                  |
+--------+----------------------+------------------+
| 라이브 스트립 (엔진 상태, 퍼널 KPI, 후보)            |
+--------------------------------------------------+
```

## 커맨드바 (상단)

화면 최상단에 위치한 정보 표시줄이다.

### 좌측

- **마켓명** (예: KRW-BTC) - 현재 선택된 코인
- **연결 상태 dot** - 녹색(연결됨) / 노란색(끊김)
- **세션 라벨** - "단타 세션" 등 현재 운영 모드

### 중앙 KPI 필

| 필 | 의미 |
|----|------|
| PnL +0원 | 오늘의 실현 손익 |
| 엔진 0/3 | 활성화된 오토파일럿 엔진 수 (최대 3개) |
| 노출 20% | 현재 포지션이 차지하는 총자산 비율 |
| 경고 0 | 시스템 경고 수 (서킷브레이커 등) |

### 우측

- **긴급 중지** - 모든 오토파일럿 엔진을 즉시 중지하고 열린 주문을 취소
- **... 메뉴** - 추가 기능 (계정 동기화, 밀도 변경, 채팅, AI Day Trader 이동)

## 마켓 사이드바 (좌측)

### 코인 검색

상단 검색창에 코인명 또는 심볼(BTC, 비트코인 등)을 입력하면 실시간 필터링된다.

### 정렬 옵션

두 개의 셀렉트박스로 정렬 기준과 방향을 선택한다:

| 정렬 기준 | 설명 |
|----------|------|
| 거래대금 | 24시간 거래대금 기준 |
| 변동률 | 24시간 가격 변동률 |
| 거래량 | 24시간 거래량 |
| 급등률 | 단기 급등 비율 |
| 추천가 승률 | AI 추천가 기준 승률 |
| 현재가 승률 | 현재가 기준 승률 |

마켓 목록에서 코인을 클릭하면 중앙 차트가 해당 코인으로 전환된다.

## 차트 영역 (중앙)

TradingView 라이트웨이트 차트를 사용한다.

### 시간 프레임 선택

차트 우측 상단 버튼으로 캔들 주기를 선택한다:

| 버튼 | 주기 |
|------|------|
| 틱 | 실시간 틱 데이터 |
| 1분 | 1분봉 |
| 10분 | 10분봉 |
| 30분 | 30분봉 (기본) |
| 60분 | 1시간봉 |
| 일 | 일봉 |

차트에는 캔들스틱과 거래량 바가 함께 표시되며, 체결된 거래가 있으면 진입/청산 마커가 차트 위에 오버레이된다.

## 액션 콘솔 (우측)

4개의 탭으로 구성된 제어 패널이다.

### 컨트롤 탭

**액션 영역:**
- **긴급 중지** - 모든 오토파일럿 즉시 중지 (빨간 버튼)
- **잔고/포지션 동기화** - 서버와 잔고 데이터 재동기화

**엔진 영역:**

3개의 오토파일럿 엔진 토글 버튼이 있다:

| 엔진 | 역할 | 기본 배분 |
|------|------|----------|
| SCALP | 초단타 (수초~수분) | 40% (12만원) |
| SWING | 중기 (수시간~수일) | 35% (10.5만원) |
| POSITION | 장기 (수일~수주) | 25% (7.5만원) |

각 버튼의 ON/OFF를 눌러 개별 엔진을 켜고 끌 수 있다. 배분 금액은 총 운영 예산의 비율이다.

**주문 옵션:**
- **시장가 폴백** - 지정가 주문이 타임아웃되면 시장가로 자동 전환 (ON/OFF)

### 리스크 탭

리스크 프리셋을 선택한다:

| 프리셋 | 설명 |
|--------|------|
| 보수 | 낮은 리스크, 낮은 수익 |
| 균형 | 중간 수준 (기본) |
| 공격 | 높은 리스크, 높은 수익 |
| 커스텀 | 사용자 직접 파라미터 설정 |

각 프리셋은 손절 비율, 익절 비율, 최대 포지션 수 등을 일괄 조정한다.

### 모니터 탭

오토파일럿 후보 큐와 이벤트 요약을 실시간으로 표시한다.

### 고급 탭

LLM 프로바이더(OpenAI/z.ai) 설정, Playwright 브라우저 연결 상태 등 개발자 수준의 설정을 관리한다.

## 라이브 스트립 (하단)

화면 하단에 고정된 얇은 바로, 오토파일럿 전체 상태를 한눈에 보여준다.

### 구성 요소 (좌→우)

1. **엔진 상태 dot** - S(SCALP), W(SWING), P(POSITION) 각각의 ON/OFF 상태를 컬러 dot으로 표시
2. **AI 헬스스코어** - 링 차트로 시스템 건강도 0~100 표시
3. **퍼널 KPI** - `매수요청 → 매수체결 → 매도요청 → 매도체결` 각 단계의 건수
4. **워커/토큰** - 활성 워커 수, LLM 토큰 잔량(%)
5. **후보 상태** - 현재 탐지된 후보 코인 요약
6. **상세 보기** - 클릭하면 딥 다이브 드로어가 열림

## 딥 다이브 드로어

라이브 스트립의 "상세 보기"를 클릭하면 화면 하단에서 슬라이드업되는 패널이다. 오토파일럿의 상세 모니터링 데이터를 탭별로 제공한다.

### 엔진 필터

드로어 우측 상단에 엔진 필터 필이 있다:
- **전체** - 모든 엔진 데이터 통합
- **초단타** - SCALP 엔진만
- **스윙** - SWING 엔진만
- **포지션** - POSITION 엔진만

### 타임라인 탭 (기본)

**상단: 커맨드 허브**

7개의 타일로 핵심 지표를 표시한다:

| 타일 | 의미 |
|------|------|
| AI 헬스스코어 | 전체 시스템 건강도 (0~100) |
| 진입 체결율 | 매수 주문 대비 체결 비율 |
| 청산 체결율 | 매도 주문 대비 체결 비율 |
| 워커/이벤트 | 활성 워커 수 / 총 이벤트 수 |
| 마지막 액션 | 가장 최근 실행된 액션 |
| 토큰 잔량 | LLM 일일 토큰 예산 잔량 |
| 액션 레이어 | 타입별 이벤트 카운트 |

**중단: 퍼널**

주문 흐름을 시각화한다:
`매수 요청 → 매수 체결 → 매도 요청 → 매도 체결 → 대기 / 취소·실패`

**하단: 3열 레이아웃**

| 열 | 내용 |
|----|------|
| 후보 | 탐지된 코인 후보 목록 (stage 뱃지 포함) |
| 타임라인 | 실시간 액션 로그 (LLM/WORKER/ORDER/PLAYWRIGHT/SYSTEM) |
| 워커 | 현재 실행 중인 워커 카드 |

**타임라인 필터:**
- 소스: 전체 / 로컬 / 서버
- 레벨: 전체 / INFO / WARN / ERROR
- 타입: 전체 / LLM / WORKER / ORDER / PLAYWRIGHT / SYSTEM
- 페이지 크기: 30 / 50 / 100

### 후보 탭

탐지된 모든 후보 코인의 상세 정보를 표시한다. 각 후보 카드에는:
- 코인명, 현재가, 변동률
- stage 뱃지 (AUTO_PASS / BORDERLINE / RULE_FAIL 등)
- 점수 미터 (기대수익률 기반)
- 커맨드 스트립 (RR, 괴리율, 기대수익 등)

### 워커 탭

활성 워커의 상태 카드를 표시한다. 각 워커는 하나의 코인에 대한 진입/청산 프로세스를 관리한다.

### 체결내역 탭

오늘의 전체 주문/체결 내역을 표시한다.

**필터:**
- 타입: 전체 / 요청 / 체결 / 취소·실패
- 소스: 전체 / 오토파일럿 / 수동·기타

각 체결 항목에는 마켓명, 이벤트 타입(BUY_REQUEST, BUY_FILLED 등), 시간, 엔진 태그가 표시된다.

### 선택단타 탭

사용자가 직접 지정한 코인(최대 8개)을 독립 루프로 반복 매매하는 모드이다.
- 전역 후보 탐색과 분리되어 독립 실행
- 기본 확인 주기: 20초
- 보유시간 규칙: 90분 경고, 120분 강제 청산

## 오토파일럿 엔진

### 작동 원리

1. **마켓 스캔** - 거래대금 상위 코인에서 기회 탐지
2. **정량 분석** - 기대수익률, RR비율, 점수 계산
3. **stage 분류**:
   - `AUTO_PASS` (점수 64+, 기대수익 0.12%+) → 즉시 진입
   - `BORDERLINE` (점수 56+, 기대수익 0.02%+) → LLM 심사 후 결정
   - `RULE_FAIL` → 진입 불가
4. **워커 생성** - 진입 결정된 코인에 워커 할당
5. **포지션 관리** - 손절/익절/트레일링 자동 실행

### 엔진별 전략

| 엔진 | 대상 | 보유 기간 | 종목 |
|------|------|----------|------|
| SCALP | 모든 코인 | 수초~수분 | 기회탐지 유니버스 |
| SWING | 주요 코인 | 수시간~수일 | BTC/ETH/SOL/XRP/ADA |
| POSITION | 주요 코인 | 수일~수주 | BTC/ETH/SOL/XRP/ADA |

### LLM 토큰 예산

일일 토큰 상한: 200,000

| 구분 | 한도 |
|------|------|
| Entry (진입 심사) | 160,000 |
| Risk Reserve (긴급) | 40,000 |

예산 초과 시 LLM 심사 없이 정량 분석만으로 운영된다 (Quant-only 폴백).

### 리스크 관리

- **일일 손실 한도**: -30,000원 도달 시 전체 엔진 정지
- **최대 동시 포지션**: 6개
- **서킷브레이커**: 연속 손실/슬리피지 감시
- **진입 실패 쿨다운**: 300초

## AI Day Trader

커맨드바 ... 메뉴에서 "AI Day Trader"를 선택하면 진입하는 별도 화면이다.

### 개요

LLM(GPT/z.ai)이 직접 시장 데이터를 분석하고, 매수/매도/홀드/대기 결정을 내리는 자율 트레이딩 모드이다. 오토파일럿의 규칙 기반 엔진과 달리, AI가 자연어 추론으로 매매 결정을 한다.

### 레이아웃

```
+--------------------------------------------------+
| 상단바: [뒤로] AI Day Trader [상태] [KPI] [시작/중지] |
+----------+--------------------+------------------+
| Top      | AI 결정 / 이벤트    | 보유 포지션        |
| Markets  | 로그 탭            | + 설정 패널        |
+----------+--------------------+------------------+
```

### 상단바

- **뒤로** - 메인 워크스페이스로 복귀
- **상태 dot** - 대기/스캔 중/AI 분석 중/주문 실행 중/일시정지/오류
- **KPI**: PnL, 거래 건수, 토큰 사용량, 보유 포지션 수

### Top Markets (좌측)

시장 스캔 결과 상위 후보 코인 목록. 각 항목에 코인명, 현재가, 변동률이 표시된다.

### AI 결정 / 이벤트 (중앙)

두 개의 탭으로 전환된다:

**AI 결정 탭:**
각 결정 카드에는:
- 액션 뱃지 (BUY: 녹색 / SELL: 빨간색 / HOLD: 노란색 / WAIT: 회색)
- 대상 마켓
- 확신도 (0~100%)
- AI의 추론 내용 (자연어)
- 시간

**이벤트 로그 탭:**
시스템 이벤트 (스캔 시작, 분석 완료, 주문 실행, 에러 등)의 시간순 로그.

### 보유 포지션 (우측 상단)

현재 AI가 보유 중인 포지션 카드. 각 카드에:
- 코인명
- 미실현 손익 (%)
- 진입가 / 현재가
- 보유 시간
- 수량

### 설정 패널 (우측 하단)

엔진 실행 전 설정하는 파라미터:

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 프로바이더 | LLM 제공사 | OpenAI |
| 모델 | 사용 모델 | 프로바이더에 따라 다름 |
| 스캔 주기 | 시장 스캔 간격 | 15초 |
| 1회 금액 | 한 번에 투자할 금액 | 10,000원 |
| 최대 포지션 | 동시 보유 가능한 포지션 수 | 3개 |
| 일일 손실한도 | 하루 최대 허용 손실 | -20,000원 |

설정은 엔진이 정지된 상태에서만 변경할 수 있다.

### 사용 방법

1. 설정 패널에서 프로바이더/모델/금액 등을 설정
2. "시작" 버튼 클릭
3. AI가 자동으로 시장 스캔 → 분석 → 매매 결정 → 주문 실행

## AI 채팅

메인 워크스페이스 우측에 슬라이드로 열리는 AI 대화 패널이다.

### 접근 방법

커맨드바 ... 메뉴에서 "채팅"을 선택하거나, 액션 콘솔 고급 탭에서 접근한다.

### 기능

- 현재 마켓에 대한 분석 질문 (예: "KRW-BTC 지금 어때?")
- 기술적 분석 요청 (RSI, MACD, 볼린저밴드)
- 매매 전략 상담
- z.ai 위임 (`/zai <질문>` 형식)

### LLM 프로바이더

| 프로바이더 | 인증 | 모델 |
|-----------|------|------|
| OpenAI | 로그인 버튼으로 인증 | GPT 계열 |
| z.ai | API 키 설정 | GLM 계열 (glm-5, glm-4.7 등) |

## 용어 사전

| 용어 | 설명 |
|------|------|
| **PnL** | Profit and Loss, 손익 |
| **KPI** | Key Performance Indicator, 핵심 성과 지표 |
| **SCALP** | 초단타 매매 (수초~수분 보유) |
| **SWING** | 스윙 매매 (수시간~수일 보유) |
| **POSITION** | 포지션 매매 (수일~수주 보유) |
| **워커** | 하나의 코인에 대한 진입/청산을 관리하는 백그라운드 프로세스 |
| **퍼널** | 주문 흐름 단계 (매수요청→매수체결→매도요청→매도체결) |
| **stage** | 후보 코인의 진입 자격 단계 |
| **AUTO_PASS** | 정량 기준 통과, 즉시 진입 가능 |
| **BORDERLINE** | 경계선, LLM 심사 필요 |
| **RULE_FAIL** | 정량 기준 미달 |
| **트레일링 스탑** | 고점 대비 일정% 하락 시 자동 청산하는 방식 |
| **서킷브레이커** | 연속 손실 시 자동으로 거래를 중단하는 안전장치 |
| **RR** | Risk-Reward ratio, 위험 대비 보상 비율 |
| **기대수익률** | 승률과 RR을 고려한 기대 수익 |
| **헬스스코어** | 시스템 전반의 건강 상태 점수 (0~100) |
| **토큰** | LLM API 호출 시 소비되는 단위 (일일 한도 관리) |
| **Quant-only 폴백** | 토큰 예산 초과 시 LLM 없이 정량 분석만으로 운영 |
| **선택 코인 루프** | 사용자 지정 코인을 독립적으로 반복 매매하는 모드 |
